phpとsapの連携をするため、saprfcのインストールを実施していますが
うまくインストールできません。
（apacheを再起動するとphp_saprfc.dllが認識されないというメッセージが
表示されます。）

誤り、不足点をご教示頂きたく、よろしくお願いいたします。
実施内容、環境は以下の通りです。

実施内容：
php.iniに以下を追加
extension=php_saprfc.dll

c:\windows\system32に以下を追加
librfc32.dll

c:\xampp\php\extに以下を追加
php_saprfc.dll （php_saprfc_530.dllをリネーム）
※php_saprfc_530.dllのままでも動作せず
　　（php.iniの名前もあわせる）

環境：
PHP: xampp1.7.2にてインストール
PHP5.3.0
OS: Windows XP SP2

私も同様の現象で悩みました。

私の場合、phpinfo() を表示しても、SAPRFCが認識されていない現象でした。

私の場合の解決策は、
・phpのバージョンとSAPRFCのバージョンを合わせること。
例) php5.2.8→php_saprfc_528.dllを使用。
・php.iniの編集
例） extension = php_saprfc_528.dll
・SAPRFCをNon Unicodeを使用する。
私の場合は、6.40のNon Unicodeのlibrfc.dllをC:\Windows\system32の下にCopy
・Microsoft Visual C++ 2008 Redistributable Package (x86)
　をインストールする。
http://www.microsoft.com/downloads/details.aspx?FamilyID=9b2da534-3e03-4391-8a4d-074b9f2bc1bf&displaylang=ja

の全てを実行して、OSリブートすることで動作しました。
